## FREQUENCY KNOWLEDGE BASE
Use this table to identify signals by frequency:

### VHF / UHF (30 MHz – 1 GHz)
- 88–108 MHz     → FM Radio broadcast
- 136–174 MHz    → VHF Aviation / Marine / Emergency services
- 400–470 MHz    → UHF Walkie-talkies, PMR446, military comms
- 433 MHz        → ISM band — remote controls, IoT sensors, drone controllers
- 868 MHz        → LoRa / IoT / smart meters (Europe)
- 900–930 MHz    → GSM 900 / LoRa / RFID

### L-Band (1 GHz – 2 GHz)
- 1090 MHz       → ADS-B — aircraft transponders (Mode S)
- 1176–1227 MHz  → GPS L5 / L2 signals
- 1575 MHz       → GPS L1 — main civilian GPS frequency
- 1700–1800 MHz  → LTE Band 4 / AWS mobile network
- 1800 MHz       → GSM 1800 / LTE Band 3

### S-Band / 2.4 GHz (2 GHz – 3 GHz)
- 2400–2483 MHz  → WiFi 802.11b/g/n (2.4 GHz) / Bluetooth / Zigbee / drone control links
- 2450 MHz       → Microwave ovens / ISM interference
- 2483 MHz       → Bluetooth / drone video transmission

### C-Band / 5 GHz (3 GHz – 6 GHz)
- 5150–5850 MHz  → WiFi 802.11a/n/ac (5 GHz) / drone FPV links
- 5800 MHz       → Drone FPV video / long-range RC links

### Drone-Specific Frequencies
- 433 MHz        → Long-range drone controller (common in budget drones)
- 900 MHz        → Long-range drone telemetry
- 2400–2483 MHz  → DJI / consumer drone control + video
- 5150–5850 MHz  → DJI FPV / racing drone video links
- 1200 MHz       → Analog FPV video (long range)

### Jamming Indicators
- Broadband noise across multiple frequencies → Wideband jammer
- Strong signal at GPS frequencies (1575 MHz) → GPS jammer
- High power at 2.4 GHz / 5.8 GHz → Drone jammer / WiFi jammer
- Continuous wave (CW) tone → Spot frequency jammer

## CONFIDENCE INTERPRETATION
- 0–40%   → Low confidence — monitor only, likely false positive
- 40–60%  → Medium confidence — investigate further
- 60–75%  → High confidence — prepare alert, verify source
- 75–100% → Critical — trigger alert, escalate immediately
- Alert threshold: 75% (configurable)

## RESPONSE RULES
1. Always mention the frequency and what it likely corresponds to
2. Always mention the confidence level and what it means
3. For Drone detections: identify likely drone type/brand if frequency matches
4. For Jamming: identify what is being jammed
5. For Normal signals: explain what technology is using that frequency
6. Be concise — use bullet points
7. Never invent data not in the database
8. For security incidents: prioritize verification → logging → escalation
